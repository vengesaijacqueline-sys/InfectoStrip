# InfectoStrip Technical Specification

## 1. Product Overview

InfectoStrip is an AI-enabled point-of-care diagnostic platform that combines paper-based biochemical testing technology with smartphone-based artificial intelligence to provide rapid diagnostic screening support in resource-constrained healthcare environments.

The platform is designed to transform biological signals generated through diagnostic reactions into objective digital interpretations that support healthcare decision-making.

The first product under development is:

# InfectoStrip UTI Plus

A urinary tract infection (UTI) screening platform designed to detect relevant urinary biomarkers through a paper-based biochemical assay and provide AI-assisted interpretation through a mobile application.

The long-term vision is to develop a diagnostic ecosystem where multiple InfectoStrip products address different disease areas through a shared digital and AI infrastructure.

Potential future applications include:

- InfectoStrip TB;
- InfectoStrip Febrile;
- InfectoStrip Maternal;
- InfectoStrip Sepsis.

---

# 2. Core Technology Components

The InfectoStrip platform consists of three primary components:

## 2.1 Paper-Based Diagnostic Strip

The physical diagnostic strip acts as the biological sensing component.

It contains specialised reaction zones designed to interact with biomarkers present within a patient sample.

When biomarkers interact with the biochemical components within the strip, measurable changes occur within specific zones.

These changes generate visual patterns, primarily through colour variation, which contain information about the biological process being screened.

The strip provides:

- rapid testing capability;
- low manufacturing complexity;
- portability;
- suitability for point-of-care environments.

---

## 2.2 Mobile Application

The InfectoStrip mobile application provides the digital interface between the user, diagnostic strip, and AI system.

The application performs several functions:

### User Guidance

The application guides healthcare workers through:

- selecting the appropriate test;
- following testing procedures;
- capturing images correctly;
- reviewing AI-assisted results.

### Image Acquisition

The application captures diagnostic strip images through a smartphone camera.

Image capture must account for:

- strip positioning;
- lighting conditions;
- image clarity;
- camera variation.

### Result Display

The application presents:

- interpreted screening results;
- confidence levels;
- warnings when image quality or model confidence is insufficient.

---

## 2.3 Artificial Intelligence System

The AI system provides automated interpretation of diagnostic strip images.

It combines computer vision and machine learning methods to analyse visual patterns generated by biochemical reactions.

The AI system consists of:

1. Image preprocessing layer.
2. Computer vision analysis layer.
3. Machine learning interpretation model.
4. Result and confidence generation layer.

---

# 3. AI Processing Pipeline

## Step 1: Image Capture

A user captures an image of the completed InfectoStrip using a smartphone.

The application collects the image while guiding the user to maintain appropriate:

- distance;
- orientation;
- lighting;
- focus.

---

## Step 2: Image Quality Assessment

Before analysis, the system evaluates whether the image is suitable for interpretation.

Potential checks include:

- image sharpness;
- brightness;
- strip alignment;
- visibility of reaction zones.

Images that do not meet quality requirements may prompt the user to retake the image.

---

## Step 3: Image Preprocessing

The system improves image consistency by applying preprocessing methods such as:

- colour correction;
- lighting normalisation;
- perspective correction;
- reaction zone identification.

This reduces variation caused by different smartphones and environmental conditions.

---

## Step 4: Feature Extraction

The AI system analyses relevant visual characteristics including:

- colour intensity;
- colour distribution;
- reaction zone patterns;
- spatial relationships between zones.

These features represent the biochemical response generated by the strip.

---

## Step 5: Machine Learning Interpretation

The machine learning model evaluates the extracted information and identifies patterns associated with screening outcomes.

The model is designed to learn from representative training data rather than relying only on fixed colour thresholds.

This improves adaptability across:

- different devices;
- different environments;
- different users.

---

## Step 6: Result Generation

The AI system generates:

- interpretation output;
- confidence score;
- uncertainty indicators.

Results are provided as clinical decision support and require healthcare professional judgement.

---

# 4. Data Requirements

Development of the AI model requires high-quality labelled datasets.

Required data includes:

## Diagnostic Images

Images of InfectoStrip tests under different conditions:

- varying lighting;
- different smartphone cameras;
- different reaction strengths;
- different image angles.

## Reference Results

Images must be linked with verified reference outcomes to allow accurate model training and evaluation.

## Environmental Variation

The dataset must reflect realistic deployment conditions, particularly those experienced in low-resource healthcare settings.
